🔗 Stream URL Settings

📖 How to Use This Player

⚠️ Important:

The CloudFront signed URL expires after approximately 1 hour. When it expires the stream will stop loading and you must refresh the URL parameters.

How to Refresh the URL Parameters

1 Open the eplus player page

Visit the original eplus player page in your browser (Chrome recommended):
https://live.eplus.jp/ex/player?ib=...
(Use the same ticket URL you originally received.)

2 Open Browser DevTools → Network tab

Press F12 (or Ctrl+Shift+I / Cmd+Option+I on Mac) to open DevTools.
Click the Network tab at the top.

📸 DevTools Network tab — filter by "m3u8" or "index.m3u8"
3 Find the .m3u8 request

In the Network tab filter box, type m3u8.
You should see a request to stream.live.eplus.jp with index.m3u8.
Click on it to see the full URL with query parameters.

📸 Click on the m3u8 request → Headers tab → Request URL
4 Copy the three CloudFront parameters

From the request URL, extract these three query parameters:

  • Policy — a long Base64-encoded string
  • Signature — a long Base64-encoded string
  • Key-Pair-Id — a short alphanumeric ID (e.g., APKAIGT7LTF4AOTKRGEQ)

Tip: Right-click the request → Copy → Copy URL, then paste it somewhere to extract the params.

5 Paste into the fields above & click "Apply"

Paste each parameter into the corresponding input field in the Stream URL Settings section above, then click ▶ Apply & Reload Player.

Alternative: Paste the Full URL

You can also paste the entire signed URL (including query string) directly into the Base URL field — the player will use it as-is when the other fields are empty.

Extracting Parameters from Cookies (Alternative Method)

CloudFront cookies can also be found in DevTools → Application tab → Cookies:

  • CloudFront-Policy → paste into Policy
  • CloudFront-Signature → paste into Signature
  • CloudFront-Key-Pair-Id → paste into Key-Pair-Id
// Example: reading cookies in DevTools Console
document.cookie.split(';').forEach(c => {
    if (c.includes('CloudFront')) console.log(c.trim());
});

Troubleshooting

  • Stream won't load? — Check that the URL parameters haven't expired (1-hour limit).
  • Black screen? — The stream may not be live yet. Wait for the scheduled start time.
  • CORS errors? — This is expected for cross-origin streams. The player handles this via HLS.js.
  • Quality not switching? — Click the ⚙️ settings icon in the player controls to select quality.
